# SYSTEM ARCHITECTURE

## Layers

### 1. Skills (Atomic)
- Single capability
- Clearly defined inputs/outputs

### 2. Playbooks (Workflow)
- Chain multiple skills
- Solve real business problems

### 3. Agents (Execution)
- Simulate roles:
  - Analyst
  - Manager
  - CFO

## Design Principles
- Modular
- Composable
- Transparent
- Auditable

## Example Flow

Business Question:
"Should we expand to a new market?"

Flow:
scenario_planning
→ financial_modeling
→ investment_decision
→ exec_storytelling
